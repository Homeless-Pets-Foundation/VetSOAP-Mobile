# v1.13.2 release + Sentry follow-ups — 2026-06-18

## Context

PR #92 merged 4 Sentry triage fixes to `main` but they were **unreleased** — production users still ran `com.captivet.mobile@1.13.1+74`, so none of the issues could drop in Sentry. This plan ships **1.13.2 to both stores**, hands the server-side `create_draft` HTTP 500 bug to the Connect team, and verifies the fixes in Sentry over the week after rollout.

Fixes already in `main` (PR #92):
- **RN-T** `secure_store_failed` — `templatePreference` SecureStore key used an illegal `:` → default-template preference never persisted + warning spam. Changed `:` → `_`.
- **RN-W** `AbortError` + 51× `HTTP_500` create_draft — broadened `isRecoverableSubmitFailure` (expected | transient | HTTP 5xx | abort) so recovered/server faults stop paging as hard errors.
- **RN-X** `AudioRecorder.record rejected` — expected native audio-session interruption → warning breadcrumb + `errorCode:'AUDIO_SESSION_INTERRUPTED'` instead of `captureException`.
- **RN-D / RN-9** auth-init watchdogs — deferred blocking cold-start `getUser` validation to lazy 401 recovery; also fixes an offline-cold-start logout bug.

Locked decisions: release **both Android + iOS store** (`production` profile); **file a GitHub issue** in `Homeless-Pets-Foundation/VetSOAP-Connect`; **daily Sentry check for ~7 days** post-release.

Caveat: EAS builds + store submits are outward-facing, billed, credential-gated; submits land in Google/Apple **review queues**.

## Steps

### 1. Version bump 1.13.1 → 1.13.2 (done)
`app.config.ts:113`, `package.json:3`, `package-lock.json` (root + `packages[""].version`). `npm install --legacy-peer-deps` synced the lock. EAS `autoIncrement` handles `versionCode`/`buildNumber`.

### 2. Pre-build
`npx expo-doctor` (auto via `.claude/hooks/pre-eas-build.sh`). Confirm EAS secrets present (`EXPO_PUBLIC_API_URL`, Supabase URL/anon, `EXPO_PUBLIC_GOOGLE_IOS_URL_SCHEME`, build-time `SENTRY_AUTH_TOKEN`).

### 3. EAS builds (store profile, both platforms)
- Android AAB: `npx --yes eas-cli@latest build --platform android --profile production --non-interactive`
- iOS IPA: `npx --yes eas-cli@latest build --platform ios --profile production --non-interactive`
- iOS pulls FFmpeg pod-source + GoogleSignIn modular-headers config plugins; on `INSTALL_PODS` failure diagnose via `~/eas-buildlog.sh <BUILD_ID>` (gzip NDJSON).

### 4. Submit to stores
- `eas submit --platform android --profile production` → Google Play internal track → promote.
- `eas submit --platform ios --profile production` → TestFlight → App Store.
- Update the clinic tablets first for fast verification.

### 5. File Connect server bug
GH issue in `Homeless-Pets-Foundation/VetSOAP-Connect` — see `docs/create_draft-http500-server-bug-2026-06-18.md`.

### 6. Sentry watch (daily ~7 days)
After 1.13.2 live + tablets updated, daily check filtering `release:com.captivet.mobile@1.13.2`: RN-T no new events; RN-X/W at warning; RN-D/9 rate drops. Compare to the baseline above.
